This update includes many many small fixes, a few big performance improvements, and one or two big new features.\r\n\r\n## Physics with Ammo.js\r\nFirst and foremost, I'm please to announce our integration with Ammo.js, a port of the Bullet physics engine. This brings to the Sandbox one of the most popular opensource physics engines on the market, and should make it possible to build much more interesting environments quickly. We've integrated many great features from the library including compound collision shapes, several types of constraints, and collision materials. You'll find on the sidebar a new tab with most of the physics options, as well as a new menu under \"Create\".\r\n\r\n![image](https://cloud.githubusercontent.com/assets/1257870/5440588/e8e13f98-8457-11e4-9bd6-a602ba5b696d.png)\r\n\r\nHere you can set the basic physical object parameters like mass or friction. For primitive objects, the collision body will exactly match the settings for the primitive. You can even link together objects using the link tool. Objects which are children of other objects become part of a compound collision.\r\n\r\n![image](https://cloud.githubusercontent.com/assets/1257870/5440691/b77d673c-8458-11e4-89c3-fc03546753de.png)\r\n\r\nThe Physics Editor will also show you a preview of the collision body - here you can see the compound collision body of several linked primitives.\r\n\r\n![image](https://cloud.githubusercontent.com/assets/1257870/5440766/3b6ab432-8459-11e4-8ed3-7e6db9d0f651.png)\r\n\r\nYou'll also get a preview for constraints. This is a hinge joining two bodies\r\n\r\nFinally, we've added an entire API for manipulating the physics on an object. Type `this.physicsAPI` for hints in the ScriptEditor.\r\n\r\n## Object Libraries\r\n\r\n![image](https://cloud.githubusercontent.com/assets/1257870/5440814/9f4fd8d8-8459-11e4-8e89-929ebb3ece18.png)\r\n\r\n\r\nIn an ongoing effort to create sample content, we've added some objects to a special new sidebar on the left. These objects will always be available while in Edit mode. Click and drag them into the scene to create one. The objects can be 3D entities, materials, behaviors or environments. We'll be adding more demo content in the future, as well as a way for you to load up libraries via a URL.\r\n\r\n## GUI Refresh\r\n![image](https://cloud.githubusercontent.com/assets/1257870/5440863/0125a4b6-845a-11e4-8cd3-12ee718ed8eb.png)\r\n\r\nThe old GUI was great, and we much prefer it to the original. Continuing with that trend, we've moved more of the theme to dark tones, and increased the number of highlight items. There are also more hover actions, more tooltips, and cleaner lines.\r\n\r\n## Lots more\r\n### Updated user data interfaces\r\nWe've moved much of the identity tracking into a property on the main scene. You can now access information about connected clients with `this.Scene.clientAPI` and `this.Scene.clients`\r\n### WebVR\r\nWe got our hands on an Oculus! If you have a WebVR compatible browser, try it out! This is a work in progress, expect more soon.\r\n### Performance improvements\r\nCheck out the commit logs from early October - we've been busting our butts to bring you that buttery smooth 60FPS you crave. We're now pooling and reusing many of the objects that are created at each frame. We've also made significant updates to the scene manager that tracks the locations of objects.\r\n### Bone Scaling\r\nBones can now scale independently of their parent scales - allowing for customization of character rigs. Check it out - open the \"Control Bone\" example, select a bone, and uncheck \"inherit parent scale\" at the Properties Window. \r\n### Updated Interpolation\r\nThe frame interpolation has been improved. You'll now see slower motion when the objects are near their proper locations, and faster when they are far. This seems to correct the small overshoot that caused some ugly single frame jitter in the past.\r\n### Active Synchronization\r\nIt's sometimes possible for the simulations to drift out of sync - especially when there are bugs! The server is not actively resetting objects over time, so even a large sync error should disappear over time.\r\n### Context Loss Restore\r\nIn some rare occasions, the browser can loose the WebGL context, and destroy all our graphics. When the context is restored, we can now rebuild our entire scene and continue on. There may be some graphics problems, but the simulation should continue on."
